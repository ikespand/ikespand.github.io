The Valhalla ([documentation](https://valhalla.readthedocs.io/en/latest/ "documentation"), [source code](https://github.com/valhalla/valhalla "source code")) is an fast, efficient, multimodal and open-source routing service which is being used by many mapping and routing giants. The engine provides an easy-to-use interface to use OpenStreetMap (OSM) data for the routing purpose. Instead of talking about the routing engine, my focus is on the map matching which is one of the several APIs offered in Valhalla. The map matching API is known as `Meili`. A great tutorial which focus on Meili can be found [here](https://towardsdatascience.com/map-matching-done-right-using-valhallas-meili-f635ebd17053). In this post, my target is to document entire process from setting up a Valhalla server to using it in Python. 
1. As usual, the docker solution is preferred one which avoids the installation of libraries on local machine thereby provides an OS independent solution. I found docker image form [gis-ops](https://github.com/gis-ops/docker-valhalla) as a good starting point. If you are on Windows, then try using PowerShell, because git-bash behaves strangely with volume mounting.
2. Pull the prebuild docker image and start the server. In case, if we need to build from the source, then documentation is [here](https://github.com/gis-ops/docker-valhalla#container-recipes "here"). 

	 1. Pull the docker images as:
	 ```bash
docker pull gisops/valhalla:latest
```

	 2. Then, we can turn the the sever for Valhalla. Below, we defined a local volume to mount i.e. where we will provide the OSM extracts and this folder is linked to the docker server.
```bash
	docker run -dt --name valhalla_2 -p 8002:8002 -v ABOSULUTE_PATH_ON_WINDOWS\valhalla\custom_files:/custom_files/ gisops/valhalla:latest
```
	 3. Check the running container by `docker ps`, and you should able to see the docker status and opened port which is 8002.
	 
	 4. At this moment, we do not have any data which we could use. Lets download OSM from the generous server of geofabrik as:
```bash
	 cd custom_files
	 wget http://download.geofabrik.de/europe/germany/baden-wuerttemberg-latest.osm.pbf
```
	 4. To use this data, we need to restart the docker as` docker restart valhalla_2`. Please note that, it can take sometime, to convert OSM data to a tar file. So, try the next steps, when you can already see a recently created/modified `valhalla_tiles.tar` file inside the `custom_files`.  For ~500MB OSm extract, it took around 30 minutes. Otherwise, you can start with smaller extracts.

3. Now, we are good to go to use the server. To test, server is running:
	1. We could try [http://localhost:8002/ ](http://localhost:8002/  "http://localhost:8002/ ") in the browser. We should see `"error_code":106`. It indicates that the server is running but we did not pass any specific API address and we could see available option.
	2. As we have upadted the data in `custom_files`, so we could query this data with the help of curl in our terminal. 
	
	```bash
curl http://localhost:8002/route --data '{"locations":[{"lat":48.632608,"lon":9.010350,"type":"break"},{"lat":48.631776,"lon":9.011528,"type":"break"}],"costing":"auto","directions_options":{"units":"miles"}}' | jq '.'
	```
